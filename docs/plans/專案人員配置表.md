# 專案人員配置表

網站專案所需的核心角色與職責分工。

## 1. 專案經理（Project Manager, PM）

- **職責：** 負責整個專案的進度掌控、預算編列、跨部門溝通，以及與客戶（或內部需求方）對接。
- **重要性：** 確保網站能如期、如質且不超支地完工。

## 2. UI/UX 設計師（User Interface / User Experience Designer）

- **UX（使用者體驗）：** 研究使用者習慣，規劃網站架構（Sitemap）、動線與線框圖（Wireframe），讓網站好用、直覺。
- **UI（使用者介面）：** 負責視覺設計，包含色彩搭配、字體、按鈕樣式、整體風格（Mockup），讓網站好看、符合品牌形象。

## 3. 前端工程師（Front-End Developer）

- **職責：** 將設計師畫好的視覺圖，轉化為瀏覽器看得懂的程式碼（HTML、CSS、JavaScript）。
- **重點：** 負責網站的互動效果、動畫，以及確保網頁在手機、平板、電腦上都能完美呈現的 **RWD 響應式設計**。

## 4. 後端工程師（Back-End Developer）

- **職責：** 負責處理網站「看不到的後台」，包含伺服器架設、資料庫設計、會員系統、金流串接、購物車邏輯及資料傳輸安全性。

## 5. SEO / GEO 專家

- **SEO（Search Engine Optimization）：** 搜尋引擎最佳化，提升網站在傳統搜尋結果的排名與曝光。
- **GEO（Generative Engine Optimization）：** 生成式搜尋引擎內容優化，讓內容更容易被 AI 生成式搜尋引用與呈現。

## 6. QA / 測試工程師（Quality Assurance Engineer）

- **職責：** 在網站上線前進行嚴格測試，找出 Bug（程式錯誤）、測試不同瀏覽器的相容性，並確保表單和交易功能運作正常。
